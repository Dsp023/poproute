# Contributing to PopRoute

Thank you for your interest in contributing to PopRoute! We welcome contributions from everyone.

## How to Contribute

### Reporting Issues

If you find any errors, typos, or areas that need improvement:

1. Check if the issue already exists in [GitHub Issues](https://github.com/Dsp023/poproute/issues)
2. If not, create a new issue with:
   - Clear title describing the problem
   - Detailed description
   - Which file(s) are affected
   - Suggested fix (if applicable)

### Suggesting Enhancements

We welcome suggestions for:
- New topics or sections
- Additional code examples
- Improved explanationsations
- Better organization
- Additional resources

Create an issue with the "enhancement" label describing your suggestion.

### Making Changes

1. **Fork the Repository**
   ```bash
   # Fork on GitHub, then clone your fork
   git clone https://github.com/YOUR-USERNAME/poproute.git
   cd poproute
   ```

2. **Create a Branch**
   ```bash
   git checkout -b feature/your-feature-name
   # or
   git checkout -b fix/your-bug-fix
   ```

3. **Make Your Changes**
   - Follow the existing documentation style
   - Keep explanations clear and concise
   - Add code examples where applicable
   - Test any code examples you add

4. **Commit Your Changes**
   ```bash
   git add .
   git commit -m "Description of your changes"
   ```

5. **Push to Your Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **Create a Pull Request**
   - Go to the original repository
   - Click "New Pull Request"
   - Select your branch
   - Describe your changes clearly

## Contribution Guidelines

### Documentation Style

- Use clear, simple language
- Explain concepts before diving into details
- Include practical examples
- Add code comments where helpful
- Link to related topics

### Code Examples

- Ensure code runs without errors
- Add comments explaining key steps
- Use consistent formatting
- Include expected output where relevant

### Markdown Formatting

- Use headers appropriately (# for main, ## for sections)
- Format code with ```python or ```bash
- Use **bold** for emphasis
- Use `backticks` for inline code or filenames
